📊 Phase 1: Creating Pretrained Models

What is Phase 1? Pre-training is the initial federated learning stage where all local models train on clean, in-distribution (ID) data without OOD detection. Models are then saved for Phase 2.

Method 1: Running Task 4 Pre-training (RECOMMENDED)

The easiest way to create pretrained models for all experiments:

# In project/main.py, modify the main block (line 608-661): TASK = 4 # Optionally set QUICK_TEST for faster execution: # QUICK_TEST = True (line 324) # Run the script: python3 main.py
⚠️ Important: Task 4 includes both pre-training and experiments. If you only want pre-training, run a single experiment instead (see Method 2).

Method 2: Running Task 2 - Federated Simulation (Alternative)

Run the basic federated learning pre-training phase:

🔬 Phase 2: Running Experiments

What is Phase 2? Experiments load the pretrained models and introduce malicious/OOD data. OOD detection (using Hyperdimensional Computing) filters out malicious updates before aggregation.

Overview of Task 4 Experiments

Task 4 includes 6 configurations:

  • Task4PreTrain: Pre-training phase (creates base models)
  • Task4Exp1: OOD detection DISABLED (baseline)
  • Task4Exp2: OOD detection ENABLED (single OOD client)
  • Task4Exp3: New OOD client with poisoned data
  • Task4Exp4: Half-poisoned client
  • Task4Exp5: Completely different dataset (animal faces)

Running All Task 4 Experiments

# In project/main.py, line 609: TASK = 4 # Optionally, control test mode (line 324): QUICK_TEST = False # Full experiments (35 pretrain rounds + 5 extra rounds) # QUICK_TEST = True # Quick test (3 pretrain rounds + 2 extra rounds) # Run: python3 main.py
⏱️ Runtime:
  • QUICK_TEST = True: ~5-10 minutes
  • QUICK_TEST = False: ~2-3 hours (full experiments)

📋 Detailed Task Guide

Task 1: Single Model Training (Baseline)

Train a CNN model on merged datasets WITHOUT federated learning.

TASK = 1 python3 main.py

Purpose: Verify the model and dataset pipeline work before moving to federated learning.

Configuration:

  • Epochs: 5
  • Batch size: 64
  • Image size: 256x256
  • All ID datasets merged together
  • Debug mode: Enabled (shows example images)

Output: Accuracy, loss, and confusion matrix plots.

Task 2: Federated Learning Phase 1 (Pre-training)

Implement federated learning with Federated Averaging aggregation.

TASK = 2 python3 main.py

Purpose: Pre-train global and local models on clean, in-distribution data.

Key Features:

  • Global model (host_id=0) and 4 local models (clients 1-4)
  • Each round: global model broadcasts → local models train → aggregation
  • Federated Averaging combines local updates
  • OOD detection is DISABLED
  • Models saved for later Phase 2 use

Configuration:

rounds 2
clients 5 (1 global + 4 local)
participants 4 (all local models participate)
save True
OOD enabled False

Output: Trained models, accuracy/loss history, plots.

Task 3: Federated Learning Phase 2 (With OOD Detection)

Load pre-trained models and enable OOD detection with a malicious client.

TASK = 3 python3 main.py

Purpose: Demonstrate OOD detection can filter out poisoned updates.

Key Features:

  • Loads models from Task 2
  • Adds 1 new malicious local model (client 5) with poisoned data
  • Enables Hyperdimensional Computing (HDC) based OOD detection
  • OOD detection uses cosine similarity to compare with global model
  • Malicious updates filtered before aggregation

Configuration:

rounds 3
clients 6 (1 global + 4 clean + 1 malicious)
participants 5
load True (from Task 2)
OOD enabled True
OOD round 2 (when malicious client joins)
OOD threshold 0.5 (cosine similarity cutoff)

Output: OOD similarity scores, filtered updates, results.

Task 4: Experimentation (5 Scenarios)

Comprehensive evaluation of OOD detection robustness across multiple scenarios.

TASK = 4 python3 main.py

Includes: Pre-training + 5 different experiments with varying OOD scenarios.

Experiment Sequence:

Step 1️⃣: Task4PreTrain (Phase 1)

  • Pre-trains for 35 rounds (or 3 in QUICK_TEST)
  • 4 clean local models + global model
  • OOD detection disabled
  • Saves models to ./.env/.saved/task4_pretrain/

Step 2️⃣: Task4Exp1 - OOD Detection DISABLED (Baseline)

  • Loads pretrained models
  • Adds 1 malicious client with complete poisoned data
  • OOD detection is OFF
  • Expected: Global accuracy drops significantly

Step 3️⃣: Task4Exp2 - OOD Detection ENABLED (Single OOD)

  • Loads pretrained models
  • Adds 1 malicious client with complete poisoned data
  • OOD detection is ON with threshold 0.5
  • Expected: Malicious client detected and filtered, accuracy stable

Step 4️⃣: Task4Exp3 - New OOD Client (Poisoned)

  • 4 clean local models + 1 new malicious client
  • Malicious client trained on full poisoned Alzheimer dataset
  • OOD detection ON
  • Expected: New OOD client detected and excluded

Step 5️⃣: Task4Exp4 - Half-Poisoned Client

  • 4 clean local models + 1 client with 50% poisoned + 50% clean data
  • Mixed data from poisoned Alzheimer + clean Pneumonia
  • OOD detection ON
  • Expected: Partial poisoning still detected as anomalous

Step 6️⃣: Task4Exp5 - Novel OOD Dataset (Animal Faces)

  • 4 clean local models + 1 client with completely different dataset
  • Animal faces from Afaces16000 dataset (entirely different domain)
  • OOD detection ON
  • Expected: Different domain easily detected as OOD

Results Output:

  • Task 4 summary: ./.env/plot/task4_summary.txt
  • Per-experiment results: ./.env/plot/task4_exp[1-5]/results.txt
  • Confusion matrices for each experiment
  • Final accuracy and loss values

Federated Configuration Example (Task 4 PreTrain)

federated_config = ConfigFederated( debug=True, # Enable debug output save=True, # Save models load=False, # Don't load pre-trained rounds=35, # 35 training rounds clients=5, # 1 global + 4 local participants=4, # 4 local models participate host_id=0, # Global model ID ood_round=36, # OOD never activates (>rounds) delete_on_load=False )

OOD Configuration Example (Exp2)

ood_config = ConfigOod( debug=True, enabled=True, # OOD detection ON hyper_size=int(1e4), # 10,000 dimensions id_client=[1,2,3,4], # Clean clients ood_client=[1], # Client 1 is malicious ood_protection=True, # Enable filtering ood_protection_thres=0.5 # Similarity threshold )

Key Configuration Parameters

Parameter Type Description
rounds int Number of federated training rounds
clients int Total clients (1 global + N local)
participants int Local models per round (excludes global)
epochs int Training epochs per round (use 1 for federated)
save bool Save trained models to disk
load bool Load pre-trained models before starting
load_round int Which round to load from (e.g., 35 for Task 4)
enabled (OOD) bool Enable OOD detection mechanism
ood_round int Round to activate OOD detection
ood_protection_thres float Cosine similarity threshold (0-1)
hyper_size int Hyperdimensional projection size (usually 1e4)

🔧 Troubleshooting

❌ Problem: Models not saving/loading

Solution:
  • Ensure ./.env/.saved/ directory exists
  • Check disk space available
  • Verify save=True in federated_config
  • For loading, ensure load_round matches saved round number

❌ Problem: Plots not generating

Solution:
  • Check that matplotlib backend is 'Agg' (for non-interactive)
  • Verify ./.env/plot/ directory exists
  • Ensure plot=True in plot_config

❌ Problem: CUDA/GPU errors

Solution:
  • The code will automatically fall back to CPU
  • For Metal (Apple Silicon), the code has built-in configuration
  • Check TensorFlow is properly installed: python3 -c "import tensorflow as tf; print(tf.config.list_physical_devices())"

❌ Problem: Out of memory

Solution:
  • Reduce batch_size in dataset_config (currently 64)
  • Reduce number of rounds
  • Use QUICK_TEST mode for initial testing

❌ Problem: Dataset download fails

Solution:
  • Check internet connection
  • Ensure Kaggle API is configured (for Kaggle datasets)
  • Try running individual dataset downloads first

💡 Tips and Best Practices

For Development/Testing:

  • Use QUICK_TEST = True while developing
  • Start with Task 1 to verify setup
  • Check .env/plot/ after each run to verify results are generated
  • Monitor console output for errors and OOD detection stats

For Production Runs:

  • Set QUICK_TEST = False for full experiments
  • Run in background if possible: nohup python3 main.py > output.log 2>&1 &
  • Monitor disk space (models can be large)
  • Save results before deleting .env directory

For Experimentation:

  • Modify ood_protection_thres (0.5 default) to test sensitivity
  • Change hyper_size to test HDC dimensionality effects
  • Adjust rounds and participants to test convergence
  • Create custom datasets for new OOD scenarios
