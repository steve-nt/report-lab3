1. Introduction
Federated Learning enables collaborative model training across distributed clients without requiring raw data to leave local devices, thereby preserving privacy. However, FL is vulnerable to Out-of-Distribution (OOD) data—malicious or anomalous inputs that can compromise the global model's integrity. This lab implements a defense strategy leveraging Hyperdimensional Computing (HDC) to identify and filter these malicious updates before aggregation.

2. Lab Environment & Design

2.1 Simulation ArchitectureThe environment is implemented sequentially to simulate multiple participants without requiring parallel training.Global Model (Server): Assigned $id=0$, it maintains the authoritative state and evaluates performance on all In-Distribution (ID) datasets.Local Models (Clients): Independent CNN models assigned unique IDs that train on specific data subsets.Datasets: The simulation utilizes Alzheimer's, Brain Tumor, and Pneumonia X-ray datasets as In-Distribution data, and a label-flipped poisoned Alzheimer's dataset or an Animal Face dataset for OOD scenarios.


2.2 Federated Learning PhasesPhase 1 (Pre-training): Global and local models are trained on ID data to establish a baseline. OOD detection is disabled.Phase 2 (OOD Detection): Malicious local models are introduced. The OOD detection mechanism is integrated to filter updates before Federated Averaging (FedAvg) aggregation.


3. Implementation Details

3.1 Task 2: Federated Learning Core (Phase 1)Initialization: Models are created using copy.deepcopy to ensure separate instances.Regression: Global weights are distributed to local models via model.set_weights(global_model.get_weights()).Aggregation (FedAvg): The system collects weights from participating local models and computes the mean for each layer to update the global model.

3.2 Task 3: OOD Detection via HDFF (Phase 2)The OOD detection mechanism follows these steps (implemented in ood/hdff.py):Projection Matrices: Unique semi-orthogonal matrices are generated once to project feature vectors into a high-dimensional space.Feature Extraction: A feedforward pass using a "dummy input" (matrix of ones) captures outputs from hidden layers without needing real client data.Bundling: Projected vectors from all layers are combined into a single "class descriptor vector" (hypervector) via superposition (addition).Security Protocol: Cosine similarity is computed between the global model's hypervector ($v_{global}$) and a local model's hypervector ($v_{local}$):Similarity $\approx 1$: The model is In-Distribution (ID) and included in aggregation.Similarity $\approx 0$: The model is Out-of-Distribution (OOD) and discarded.

4. Experimental Results and Analysis

4.1 Experiment 1: OOD Detection DisabledSetup: 1 OOD local model training on the fully poisoned Balzheimer5100_poisoned() dataset.Observation: Without detection, the global model's accuracy deteriorates as it aggregates malicious weights.

4.2 Experiment 2 & 3: OOD Detection EnabledSetup: 1 OOD model (Client 5) introduced alongside ID models (Clients 1-4).Observation: The system successfully identifies Client 5 as OOD because its cosine similarity falls below the defined threshold (e.g., 0.7 or 0.5).Impact: The global model's accuracy remains stable as malicious updates are excluded from aggregation.

4.3 Experiment 5: New OOD DatasetSetup: Client 5 trains on the Afaces16000 (Animal Faces) dataset.Observation: Even with non-poisoned but entirely different data distributions, the HDFF signature correctly identifies the mismatch against the global model's medical imaging distribution.

5. ConclusionThe implementation demonstrates that Hyperdimensional Feature Fusion (HDFF) is an effective, privacy-preserving method for OOD detection in Federated Learning. By comparing the internal structural "signatures" of models rather than raw data, the system successfully filters malicious and anomalous updates, maintaining the robustness of the global model.
